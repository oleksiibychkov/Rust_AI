# Розділ 12. Практикум: Агент-патрульний на сітці

## Анотація

Це фінальний розділ Частини I. Ви пройшли шлях від змінних та типів (Розділ 5) через умови, цикли, масиви, рядки та ввід до функцій (Розділ 11). Кожна тема вивчалась окремо, з ізольованими прикладами. Тепер час об'єднати все в одному працюючому проєкті — текстовій симуляції БПЛА, що патрулює прямокутну сітку. Агент має координати, енергію, напрямок, патрульний маршрут. Він приймає команди від оператора, виконує їх, витрачає енергію, і повертається на базу, коли заряд критичний. Весь код — в одному `main.rs`, без структур, без ownership, без модулів. Чистий імперативний Rust.

## 12.1. Постановка задачі

Ви будуєте текстову симуляцію автономного агента на двовимірній сітці розміром 20 x 20 клітинок. Агент починає на базі (координати 0, 0) з повним зарядом батареї (100%). Оператор вводить команди з клавіатури. Агент виконує команди, витрачає енергію на кожну дію, і може повернутись на базу для зарядки.

Вимоги до симуляції:

Агент має координати (x, y), рівень батареї (0–100), поточний курс (0–359 градусів), та логічний прапорець "у повітрі чи на землі". Кожен рух витрачає 5% батареї. Поворот витрачає 2%. Зліт та посадка — по 10%. Коли батарея менше 15% — агент автоматично повертається на базу. Агент не може вийти за межі сітки (координати від 0 до 19).

Команди оператора: "зліт", "посадка", "вперед", "назад", "ліво", "право", "курс" (з параметром), "статус", "заряд", "вихід".

### Які знання з попередніх розділів знадобляться

| Розділ | Що використовуємо |
|--------|------------------|
| 5. Змінні | `let mut` для стану агента, типи `f64`, `u8`, `u16`, `bool` |
| 6. Умови | `if`/`else` для логіки рішень, перевірки батареї, меж сітки |
| 7. Цикли | `loop` для головного циклу, `for` для маршруту |

| 11. Функції | Декомпозиція: кожна команда — окрема функція |

## 12.2. Етап 1: Стан агента та константи

Почнемо з оголошення всіх констант та змінних стану. На цьому етапі ніяких функцій — лише дані.

```rust
use std::io;

// Константи симуляції
const GRID_SIZE: i32 = 20;
const BATTERY_FULL: u8 = 100;
const BATTERY_CRITICAL: u8 = 15;
const COST_MOVE: u8 = 5;
const COST_TURN: u8 = 2;
const COST_TAKEOFF: u8 = 10;
const COST_LAND: u8 = 10;

fn main() {
    // Стан агента — набір окремих змінних
    let mut pos_x: i32 = 0;
    let mut pos_y: i32 = 0;
    let mut battery: u8 = BATTERY_FULL;
    let mut heading: u16 = 0;       // градуси: 0=північ, 90=схід
    let mut airborne: bool = false;
    let mut mission_active: bool = true;

    println!("=== Симуляція БПЛА-агента ===");
    println!("Сітка {}x{}, база в (0, 0)", GRID_SIZE, GRID_SIZE);
    println!("Введіть 'допомога' для списку команд.\n");
}
```

Стан агента — це сім окремих змінних. Це незручно (у Частині II ми об'єднаємо їх у структуру), але для імперативного коду — нормальний підхід. Константи виносимо за `main`, бо вони глобальні для програми і не змінюються.

## 12.3. Етап 2: Функції читання та допоміжні

Виділяємо перші функції — ті, що не залежать від стану агента.

```rust
/// Зчитує команду з клавіатури, повертає очищений рядок
fn read_command() -> String {
    print!("> ");
    // Примусовий вивід буфера (без \n print не завжди виводиться)
    use std::io::Write;
    io::stdout().flush().unwrap();

    let mut input = String::new();
    io::stdin().read_line(&mut input).expect("Помилка читання");
    input.trim().to_lowercase()
}

/// Обмежує координату межами сітки [0, GRID_SIZE-1]
fn clamp_coord(value: i32) -> i32 {
    if value < 0 { 0 }
    else if value >= GRID_SIZE { GRID_SIZE - 1 }
    else { value }
}

/// Текстовий опис рівня батареї
fn battery_label(level: u8) -> &'static str {
    if level < 15 { "КРИТИЧНИЙ" }
    else if level < 30 { "низький" }
    else if level < 60 { "нормальний" }
    else { "повний" }
}

/// Текстовий опис напрямку за градусами
fn heading_name(deg: u16) -> &'static str {
    if deg < 45 || deg >= 315 { "Пн" }
    else if deg < 135 { "Сх" }
    else if deg < 225 { "Пд" }
    else { "Зх" }
}

/// Виводить список доступних команд
fn print_help() {
    println!("Команди:");
    println!("  зліт     — підняти агента в повітря");
    println!("  посадка   — посадити агента");
    println!("  вперед    — рух на одну клітинку за курсом");
    println!("  назад     — рух на одну клітинку проти курсу");
    println!("  ліво      — поворот на 90° ліворуч");
    println!("  право     — поворот на 90° праворуч");
    println!("  статус    — показати стан агента");
    println!("  заряд     — зарядити батарею (тільки на базі)");
    println!("  допомога  — цей список");
    println!("  вихід     — завершити симуляцію");
}
```

Кожна функція має одну відповідальність. `read_command` — лише читання. `clamp_coord` — лише обмеження числа. `battery_label` — лише текстовий опис. Жодна з них не змінює стан агента — вони або повертають нове значення, або виводять текст.

Зверніть увагу на `use std::io::Write;` всередині функції `read_command`. Це потрібно для методу `flush()`, який примусово виводить буфер. Без нього `print!("> ")` (без переносу рядка) може не з'явитись на екрані до наступного `println!`.

## 12.4. Етап 3: Функції стану

Функції, що виводять поточний стан агента. Вони приймають дані лише для читання — за значенням (Copy-типи) та `&str`:

```rust
/// Вивід повного статусу агента
fn print_status(x: i32, y: i32, battery: u8, heading: u16, airborne: bool) {
    println!("--- Статус БПЛА ---");
    println!("  Позиція: ({}, {})", x, y);
    println!("  Курс: {}° ({})", heading, heading_name(heading));
    println!("  Батарея: {}% [{}]", battery, battery_label(battery));
    let state = if airborne { "у повітрі" } else { "на землі" };
    println!("  Стан: {}", state);
    if x == 0 && y == 0 {
        println!("  [На базі]");
    }
}
```

Функція `print_status` має 5 параметрів — це на межі. У Частині II ми вирішимо це структурою `Agent`, але зараз — працюємо з тим, що маємо. Всі параметри — Copy-типи, тому передаються за значенням (копією). Функція лише читає та виводить, нічого не змінює.

## 12.5. Етап 4: Функції модифікації стану

Функції, що змінюють стан агента. Вони приймають `&mut` для тих змінних, які повинні змінитись:

```rust
/// Витратити енергію. Повертає true, якщо вистачило.
fn consume_battery(battery: &mut u8, cost: u8) -> bool {
    if *battery >= cost {
        *battery = battery.saturating_sub(cost);
        true
    } else {
        println!("  Недостатньо батареї! Потрібно {}%, є {}%", cost, battery);
        false
    }
}

/// Зліт: перевіряє умови, змінює стан
fn cmd_takeoff(airborne: &mut bool, battery: &mut u8) {
    if *airborne {
        println!("  Агент вже в повітрі!");
        return;
    }
    if consume_battery(battery, COST_TAKEOFF) {
        *airborne = true;
        println!("  Зліт виконано! Батарея: {}%", battery);
    }
}

/// Посадка
fn cmd_land(airborne: &mut bool, battery: &mut u8) {
    if !*airborne {
        println!("  Агент вже на землі!");
        return;
    }
    if consume_battery(battery, COST_LAND) {
        *airborne = false;
        println!("  Посадка виконана! Батарея: {}%", battery);
    }
}

/// Рух вперед за курсом
fn cmd_forward(x: &mut i32, y: &mut i32, heading: u16, airborne: bool, battery: &mut u8) {
    if !airborne {
        println!("  Спершу виконайте зліт!");
        return;
    }
    if !consume_battery(battery, COST_MOVE) {
        return;
    }
    // Обчислити зміщення за курсом
    let (dx, dy) = direction_delta(heading);
    *x = clamp_coord(*x + dx);
    *y = clamp_coord(*y + dy);
    println!("  Рух вперед → ({}, {}). Батарея: {}%", x, y, battery);
}

/// Рух назад (проти курсу)
fn cmd_backward(x: &mut i32, y: &mut i32, heading: u16, airborne: bool, battery: &mut u8) {
    if !airborne {
        println!("  Спершу виконайте зліт!");
        return;
    }
    if !consume_battery(battery, COST_MOVE) {
        return;
    }
    let (dx, dy) = direction_delta(heading);
    *x = clamp_coord(*x - dx);
    *y = clamp_coord(*y - dy);
    println!("  Рух назад → ({}, {}). Батарея: {}%", x, y, battery);
}

/// Обчислити зміщення (dx, dy) за курсом
fn direction_delta(heading: u16) -> (i32, i32) {
    if heading < 45 || heading >= 315 { (0, 1) }      // Пн: y+1
    else if heading < 135 { (1, 0) }                    // Сх: x+1
    else if heading < 225 { (0, -1) }                    // Пд: y-1
    else { (-1, 0) }                                     // Зх: x-1
}

/// Поворот ліворуч (курс -90°)
fn cmd_turn_left(heading: &mut u16, airborne: bool, battery: &mut u8) {
    if !airborne {
        println!("  Спершу виконайте зліт!");
        return;
    }
    if !consume_battery(battery, COST_TURN) {
        return;
    }
    *heading = (*heading + 270) % 360; // -90 через модульну арифметику
    println!("  Поворот ліворуч → {}° ({}). Батарея: {}%",
             heading, heading_name(*heading), battery);
}

/// Поворот праворуч (курс +90°)
fn cmd_turn_right(heading: &mut u16, airborne: bool, battery: &mut u8) {
    if !airborne {
        println!("  Спершу виконайте зліт!");
        return;
    }
    if !consume_battery(battery, COST_TURN) {
        return;
    }
    *heading = (*heading + 90) % 360;
    println!("  Поворот праворуч → {}° ({}). Батарея: {}%",
             heading, heading_name(*heading), battery);
}

/// Зарядка: тільки на базі та на землі
fn cmd_charge(x: i32, y: i32, airborne: bool, battery: &mut u8) {
    if airborne {
        println!("  Неможливо зарядити в повітрі! Виконайте посадку.");
        return;
    }
    if x != 0 || y != 0 {
        println!("  Зарядка доступна тільки на базі (0, 0). Ваша позиція: ({}, {})", x, y);
        return;
    }
    *battery = BATTERY_FULL;
    println!("  Батарея заряджена до 100%!");
}
```

Зверніть увагу на патерн guard clauses: кожна функція спершу перевіряє умови (`if !airborne`, `if !consume_battery(...)`) і повертається рано з `return`, якщо умова не виконана. Основна логіка — в кінці функції, без вкладених `if`.

Також зверніть увагу на вибір `&mut` vs значення. `heading: u16` у `cmd_forward` передається за значенням, бо функція лише читає курс. `heading: &mut u16` у `cmd_turn_left` — бо функція змінює курс.

## 12.6. Етап 5: Головний цикл

Тепер збираємо все разом у `main`:

```rust
fn main() {
    let mut pos_x: i32 = 0;
    let mut pos_y: i32 = 0;
    let mut battery: u8 = BATTERY_FULL;
    let mut heading: u16 = 0;
    let mut airborne: bool = false;

    println!("=== Симуляція БПЛА-агента ===");
    println!("Сітка {}x{}, база (0,0), батарея {}%", GRID_SIZE, GRID_SIZE, battery);
    println!("Введіть 'допомога' для списку команд.\n");

    loop {
        // Перевірка критичного заряду
        if battery < BATTERY_CRITICAL && airborne {
            println!("\n  [УВАГА] Батарея критична ({}%)! Аварійна посадка!", battery);
            airborne = false;
            println!("  Поверніться на базу для зарядки.\n");
        }

        let cmd = read_command();

        if cmd == "вихід" {
            println!("\nСимуляцію завершено.");
            println!("Фінальна позиція: ({}, {})", pos_x, pos_y);
            println!("Залишок батареї: {}%", battery);
            break;
        }

        if cmd == "допомога" {
            print_help();
        } else if cmd == "статус" {
            print_status(pos_x, pos_y, battery, heading, airborne);
        } else if cmd == "зліт" {
            cmd_takeoff(&mut airborne, &mut battery);
        } else if cmd == "посадка" {
            cmd_land(&mut airborne, &mut battery);
        } else if cmd == "вперед" {
            cmd_forward(&mut pos_x, &mut pos_y, heading, airborne, &mut battery);
        } else if cmd == "назад" {
            cmd_backward(&mut pos_x, &mut pos_y, heading, airborne, &mut battery);
        } else if cmd == "ліво" {
            cmd_turn_left(&mut heading, airborne, &mut battery);
        } else if cmd == "право" {
            cmd_turn_right(&mut heading, airborne, &mut battery);
        } else if cmd == "заряд" {
            cmd_charge(pos_x, pos_y, airborne, &mut battery);
        } else if cmd.is_empty() {
            // порожній рядок — ігноруємо
        } else {
            println!("  Невідома команда: '{}'. Введіть 'допомога'.", cmd);
        }
    }
}
```

Функція `main` стала диспетчером: зчитує команду, визначає, яку функцію викликати, передає потрібні змінні. Жодної обчислювальної логіки — лише маршрутизація.

Перевірка критичного заряду на початку циклу — це guard clause для всього циклу. Якщо батарея менша за 15% і агент у повітрі — автоматична посадка, незалежно від команди оператора. Це поведінка "автопілота" — агент сам приймає рішення про безпеку.

## 12.7. Аналіз коду: що ми використали

Подивимось, як знання з кожного розділу втілились у цій програмі:

Змінні (Розділ 5): `let mut pos_x: i32`, `let mut battery: u8`, `let mut airborne: bool` — стан агента як набір типізованих змінних. Константи `const GRID_SIZE: i32 = 20` для параметрів симуляції.

Умови (Розділ 6): `if !airborne { return; }` — guard clauses у кожній команді. `if battery < BATTERY_CRITICAL` — автоматичне рішення агента. `if`-вирази для обчислень: `let state = if airborne { "у повітрі" } else { "на землі" }`.

Цикли (Розділ 7): `loop { ... break; }` — нескінченний цикл з виходом за командою "вихід". Це головний цикл симуляції — серце програми.

Масиви/кортежі (Розділ 8): Кортеж `(i32, i32)` для повернення зміщення з `direction_delta`. Деструктуризація `let (dx, dy) = direction_delta(heading)`.

Рядки (Розділ 9): `input.trim().to_lowercase()` — очистка та нормалізація вводу. Порівняння `cmd == "зліт"` — рядки як ідентифікатори команд.

Ввід (Розділ 10): `io::stdin().read_line(&mut input)` — зчитування команд. `io::stdout().flush()` — примусовий вивід промпту.

Функції (Розділ 11): 14 функцій із чітким розподілом відповідальностей. Передача за значенням для читання, `&mut` для модифікації. Guard clauses для валідації. Повернення `bool` для індикації успіху.

## 12.8. Обмеження та погляд у Частину II

Цей код працює, але має очевидні обмеження, які стануть мотивацією для наступних розділів:

Стан агента — 7 окремих змінних. При додаванні нових полів (висота, швидкість, список цілей) кількість параметрів у функціях стане некерованою. Рішення — структури (Розділ 17, Частина II).

Ланцюжок `else if` для команд — якщо команд стане 30, код буде нечитабельним. Рішення — enum та match (Розділ 18, Частина II).

Немає збереження історії — агент не пам'ятає, де він був. Рішення — Vec (Розділ 22, Частина II).

Немає обробки помилок — `unwrap()` та `expect()` паніку при будь-якій проблемі. Рішення — Result та Option (Розділ 24, Частина II).

Все в одному файлі — 200+ рядків в одному `main.rs`. Рішення — модулі (Розділ 19, Частина II).

Ці обмеження — не баги, а природні межі імперативного підходу. У Частині II ви навчитесь знімати кожне з них.

## Troubleshooting

### Промпт `> ` не з'являється перед введенням

Причина: `print!("> ")` без переносу рядка не виводиться, бо буфер не пустий. Додайте:
```rust
use std::io::Write;
io::stdout().flush().unwrap();
```

### Команда не розпізнається, хоча введено правильно

Причина: пробіли або різний регістр. Переконайтесь, що використовуєте `input.trim().to_lowercase()`.

### Координати виходять за межі

Причина: не застосовується обмеження після зміни. Завжди використовуйте `clamp_coord()` після кожної зміни координат.

### `cannot borrow as mutable more than once`

Причина: ви передаєте `&mut battery` та ще одну `&mut` змінну, що залежить від `battery`. Rust не дозволяє два `&mut` одночасно. Рішення: збережіть потрібне значення у локальній змінній перед викликом.

### Зрушення курсу: -90° дає від'ємне число

Причина: `heading - 90` для `u16` при `heading = 0` дає переповнення. Рішення: використовуйте `(heading + 270) % 360` замість `heading - 90`.

## Резюме

Ви створили першу повноцінну інтерактивну програму на Rust, об'єднавши всі концепції Частини I. Головний цикл `loop` зчитує команди, ланцюжок `if`/`else` маршрутизує до відповідних функцій, кожна функція перевіряє умови через guard clauses і змінює стан через `&mut`.

Ключовий навик цього розділу — декомпозиція. Замість 200 рядків у `main` — 14 функцій по 5–15 рядків кожна. Кожна тестована окремо, кожна зрозуміла без контексту решти програми.

Ви також побачили обмеження імперативного підходу: 7 окремих змінних для стану, 5+ параметрів у функціях, ланцюжок `else if` для 10 команд. Ці обмеження — природна мотивація для Частини II, де ви вивчите ownership, структури, enum, модулі та інші інструменти, що зроблять код елегантнішим та надійнішим.
