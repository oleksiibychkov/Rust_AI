Два попередніх розділи готували ґрунт: ви дізнались, як працює комп'ютер і компілятор, встановили Rust та налаштували середовище. Тепер — перший код. ## Ключові терміни

**fn** — ключове слово для оголошення функції (від англійського function).

**main** — спеціальна функція, з якої починається виконання програми. Точка входу.

**Macro (макрос)** — спеціальна конструкція мови, що генерує код під час компіляції. У Rust макроси позначаються знаком оклику: `println!`, `format!`, `vec!`.

**println!** — макрос для виводу тексту на екран з переходом на новий рядок.

**String literal (рядковий літерал)** — текст у подвійних лапках: `"Hello"`.

**Format string (рядок форматування)** — рядок із заповнювачами `{}`, куди підставляються значення.

**Statement (інструкція)** — одиниця коду, що виконує дію і закінчується крапкою з комою.

**Comment (коментар)** — текст у коді, який компілятор ігнорує. Для людей, не для машини.

## Мотиваційний кейс

Перша програма "Hello, world!" — це не просто традиція. Вона з'явилась у 1978 році в книзі Браяна Кернігана та Денніса Рітчі "The C Programming Language" і відтоді стала стандартним тестом: чи працює середовище, чи компілюється код, чи бачимо ми результат. Але для БПЛА "Hello, world!" — це занадто мало. Бортовий комп'ютер дрона повинен постійно виводити діагностичну інформацію: координати, висоту, заряд батареї, стан місії. Якщо оператор не бачить цих даних — він "сліпий". Тому перше, що має вміти програма нашого агента — це зрозуміло, структуровано вивести свій стан на екран. Саме цьому ми навчимось у цьому розділі.

## 3.1. Анатомія програми: fn main()

Відкрийте файл `src/main.rs` у проєкті, створеному в попередньому розділі. Ось що там:

```rust
fn main() {
    println!("Hello, world!");
}
```

Три рядки. Перша повноцінна програма на Rust. Розберемо кожен символ.

```
fn main() {
│  │    │  │
│  │    │  └── Відкриваюча фігурна дужка: початок тіла функції
│  │    └───── Порожні круглі дужки: функція не приймає параметрів
│  └────────── main: ім'я функції (точка входу програми)
└───────────── fn: ключове слово "function" (оголосити функцію)
```

**`fn`** — скорочення від function. Це ключове слово повідомляє компілятору: "тут починається оголошення функції". Функція — це іменований блок коду, який можна викликати. У Частині I ми працюватимемо з однією функцією — `main`. У Розділі 11 ви навчитесь створювати власні функції.

**`main`** — ім'я цієї конкретної функції. Назва `main` не випадкова і не може бути іншою. Коли операційна система запускає вашу програму, вона шукає функцію з ім'ям `main` і починає виконання саме з неї. Якщо функції `main` немає — програма не скомпілюється. Це точка входу (entry point).

**`()`** — круглі дужки після імені. Вони означають, що функція може приймати вхідні дані (параметри). Порожні дужки означають: ця функція нічого не приймає. Пізніше, коли ви писатимете функції на кшталт `calculate_distance(x1, y1, x2, y2)`, між дужками будуть параметри.

**`{}`** — фігурні дужки обмежують тіло функції: все, що між `{` та `}`, виконується при виклику цієї функції. Відкриваюча дужка `{` стоїть на тому самому рядку, що й `fn main()` — це стандартний стиль Rust (і `cargo fmt` виправить, якщо ви поставите дужку на наступний рядок).

**`println!("Hello, world!");`** — це власне дія: вивести текст на екран. Розберемо:

- `println!` — макрос виводу. Знак оклику `!` — не для емоцій. Він повідомляє компілятору, що це макрос, а не звичайна функція. Яка різниця? Макрос може приймати різну кількість аргументів (`println!("{}", x)`, `println!("{} {}", x, y)`, `println!("{} {} {}", x, y, z)`), а звичайна функція у Rust має фіксовану кількість параметрів. Глибше ми розглянемо макроси у Частині VI, а поки запам'ятайте: бачите `!` після імені — це макрос.

- `"Hello, world!"` — рядковий літерал (string literal). Текст у подвійних лапках. Саме цей текст з'явиться на екрані.

- `;` — крапка з комою завершує інструкцію (statement). У Rust майже кожен рядок коду, що виконує дію, закінчується крапкою з комою. Якщо забудете — компілятор підкаже. "Майже" — тому що є винятки, пов'язані з виразами (expressions), але про це поговоримо у Розділі 11.

Запустіть програму:

```
cargo run
```

Вивід:

```
Hello, world!
```

Що відбулося: cargo викликав rustc, rustc перетворив ваш main.rs на виконуваний файл, cargo запустив цей файл, функція `main` виконала єдину інструкцію — `println!`, яка вивела текст на екран, після чого програма завершилась.

## 3.2. Вивід тексту: println! та print!

`println!` — не єдиний спосіб вивести текст. Є ще `print!`. Різниця проста: `println!` додає перехід на новий рядок після тексту, `print!` — ні.

Наступна програма демонструє різницю між `print!` та `println!`. Зверніть увагу, як текст із `print!` з'являється на одному рядку, а `println!` завершує рядок і переходить на наступний.

```rust
fn main() {
    print!("Один ");
    print!("Два ");
    print!("Три");
    println!(); // порожній println! — просто перехід на новий рядок
    println!("Чотири");
    println!("П'ять");
}
```

Вивід:

```
Один Два Три
Чотири
П'ять
```

Три виклики `print!` склали текст в один рядок, бо `print!` не додає перехід на новий рядок. Потім `println!()` без аргументів просто перейшов на новий рядок. Після цього кожен `println!` виводить текст і одразу переходить на новий рядок.

Назва `println` розшифровується як "print line" — "надрукувати рядок". Літера `ln` на кінці — це скорочення від line.

## 3.3. Форматований вивід: підставляємо значення

Виводити фіксований текст — це лише початок. Програми працюють з даними, і ці дані потрібно показувати. Для цього `println!` підтримує рядки форматування (format strings) — рядки із заповнювачами `{}`, замість яких підставляються значення.

Наступний приклад показує, як вивести числове значення всередині тексту. Заповнювач `{}` буде замінений на значення, передане після рядка.

```rust
fn main() {
    println!("Два плюс два дорівнює {}", 2 + 2);
}
```

Вивід:

```
Два плюс два дорівнює 4
```

Заповнювач `{}` означає: "підстав сюди наступне значення". Rust обчислив вираз `2 + 2`, отримав `4`, і вставив його замість `{}`.

Можна підставляти кілька значень — кожне відповідає своєму `{}` зліва направо. Ось програма, що виводить координати точки:

```rust
fn main() {
    println!("Координати: x = {}, y = {}", 10, 25);
}
```

Вивід:

```
Координати: x = 10, y = 25
```

Перший `{}` замінився на `10`, другий — на `25`. Порядок має значення: значення підставляються по черзі.

Можна підставляти не лише числа, а й текст, результати обчислень, і навіть інші рядки:

```rust
fn main() {
    println!("Привіт, {}! Тобі {} років.", "Оксано", 19);
    println!("Площа: {} кв.м", 5 * 8);
    println!("{} + {} = {}", 100, 200, 100 + 200);
}
```

Вивід:

```
Привіт, Оксано! Тобі 19 років.
Площа: 40 кв.м
100 + 200 = 300
```

Кількість заповнювачів `{}` має збігатися з кількістю значень після коми. Якщо заповнювачів більше, ніж значень, або навпаки — компілятор видасть помилку. Це одна з речей, яку Rust перевіряє під час компіляції, на відміну від деяких інших мов, де така невідповідність виявляється лише при запуску.

## 3.4. Точність дробових чисел: {:.N}

Дробові числа (числа з крапкою) часто потрібно виводити з певною точністю. GPS-координати БПЛА мають шість знаків після крапки. Заряд батареї у відсотках — один. Температура — два. Для цього є формат `{:.N}`, де N — кількість цифр після десяткової крапки.

Наступний приклад показує, як одне й те саме число виводиться з різною точністю:

```rust
fn main() {
    let pi = 3.141592653589793;

    println!("Без форматування: {}", pi);
    println!("Два знаки:        {:.2}", pi);
    println!("Чотири знаки:     {:.4}", pi);
    println!("Нуль знаків:      {:.0}", pi);
}
```

Вивід:

```
Без форматування: 3.141592653589793
Два знаки:        3.14
Чотири знаки:     3.1416
Нуль знаків:      3
```

Тут ми вперше використали `let` — ключове слово для створення змінної. Рядок `let pi = 3.141592653589793;` створює змінну з іменем `pi` і присвоює їй значення. Ми детально розберемо змінні у Розділі 5, а поки достатньо розуміти: `let ім'я = значення;` створює іменоване значення, яке можна використовувати далі.

Зверніть увагу на `{:.4}` — число 3.141592... було округлено до 3.1416, а не обрізано до 3.1415. Rust округлює за стандартними математичними правилами.

Для координат БПЛА це особливо корисно. GPS дає координати з високою точністю, але для відображення оператору зазвичай достатньо шести знаків (точність до ~11 сантиметрів):

```rust
fn main() {
    let latitude = 50.45000123456789;
    let longitude = 30.52340987654321;

    // Повна точність — занадто багато цифр для людини
    println!("Повна точність: {}, {}", latitude, longitude);

    // Шість знаків — достатня точність для навігації
    println!("Навігаційна точність: {:.6}, {:.6}", latitude, longitude);

    // Два знаки — грубе розташування (місто)
    println!("Грубе розташування: {:.2}, {:.2}", latitude, longitude);
}
```

Вивід:

```
Повна точність: 50.45000123456789, 30.52340987654321
Навігаційна точність: 50.450001, 30.523410
Грубе розташування: 50.45, 30.52
```

## 3.5. Налагоджувальний вивід: {:?}

Окрім `{}`, є ще один заповнювач: `{:?}`. Він виводить значення у налагоджувальному (debug) форматі. Різниця в тому, що `{}` призначений для кінцевого користувача (красиво, зрозуміло), а `{:?}` — для розробника (максимум інформації, навіть якщо не дуже гарно).

Для простих чисел різниця незначна. Але для рядків вона помітна:

```rust
fn main() {
    println!("Display: {}", "hello");
    println!("Debug:   {:?}", "hello");
}
```

Вивід:

```
Display: hello
Debug:   "hello"
```

У debug-форматі рядок виводиться в лапках — щоб було видно, що це саме рядок, а не щось інше. Це буде дуже корисно пізніше, коли ви працюватимете зі структурами даних: `{:?}` покаже вміст структури цілком, а `{}` може навіть не працювати без додаткових налаштувань.

Є ще `{:#?}` — "гарний" debug-формат, який додає відступи та переноси рядків для складних структур. Ми повернемося до нього у Розділі 14, коли з'являться власні структури даних.

## 3.6. Коментарі: нотатки для людей

Код пишуть для двох аудиторій: для компілятора і для людей. Компілятор потребує точних інструкцій. Люди — пояснень, контексту, причин. Для цього існують коментарі — текст у коді, який компілятор повністю ігнорує.

Rust підтримує три види коментарів.

**Однорядковий коментар** починається з `//` і продовжується до кінця рядка:

```rust
fn main() {
    // Це коментар. Компілятор його ігнорує.
    println!("А цей рядок виконується"); // коментар може бути і після коду
}
```

Вивід:

```
А цей рядок виконується
```

Компілятор пропустив обидва коментарі — вони існують лише для людини, що читає код.

**Багаторядковий коментар** починається з `/*` і закінчується `*/`:

```rust
fn main() {
    /* 
       Цей коментар займає кілька рядків.
       Корисно для довгих пояснень або
       тимчасового вимкнення блоку коду.
    */
    println!("Програма працює");
}
```

На практиці однорядкові коментарі `//` використовуються значно частіше. Багаторядкові `/* */` зазвичай застосовують лише для тимчасового "вимкнення" великого блоку коду під час налагодження.

**Документаційний коментар** починається з `///` (три слеші) і використовується для документації функцій, структур, модулів. Ми повернемось до нього у Розділі 18, коли вивчатимемо модулі. Поки просто знайте, що він існує:

```rust
/// Головна функція програми.
/// Виводить привітання на екран.
fn main() {
    println!("Привіт!");
}
```

Коли і що коментувати? Хороший коментар пояснює *чому*, а не *що*. Порівняйте:

```rust
// Неефективний коментар — і так видно, що виводиться текст:
println!("Заряд: 75%"); // виводимо заряд батареї

// Корисний коментар — пояснює причину рішення:
println!("Заряд: 75%"); // поріг повернення: 30%, є запас
```

Перший коментар дублює те, що і так очевидно з коду. Другий пояснює контекст, якого з самого коду не видно.

## 3.7. Спеціальні символи в рядках

Іноді потрібно вивести текст, що містить "спеціальні" символи: подвійні лапки, зворотний слеш, або перехід на новий рядок посеред тексту. Для цього використовуються escape-послідовності (послідовності екранування) — комбінації зі зворотним слешем `\`.

Наступний приклад показує найчастіше використовувані escape-послідовності:

```rust
fn main() {
    // \n — перехід на новий рядок
    println!("Перший рядок\nДругий рядок\nТретій рядок");

    // \t — табуляція (відступ)
    println!("Ім'я:\tОлег\nВік:\t21");

    // \" — подвійна лапка всередині рядка
    println!("Він сказав: \"Привіт!\"");

    // \\ — сам зворотний слеш
    println!("Шлях: C:\\Users\\student\\projects");
}
```

Вивід:

```
Перший рядок
Другий рядок
Третій рядок
Ім'я:	Олег
Вік:	21
Він сказав: "Привіт!"
Шлях: C:\Users\student\projects
```

Escape-послідовності обробляються компілятором: у виконуваному файлі `\n` перетворюється на реальний перехід на новий рядок, `\t` — на реальну табуляцію. Зверніть увагу: щоб вивести один зворотний слеш `\`, потрібно написати два `\\`, тому що `\` — це початок escape-послідовності.

## 3.8. Практика: статус БПЛА-агента

Тепер об'єднаємо все, що вивчили, у першу програму наскрізного проєкту. Наш БПЛА-агент повинен повідомляти свій статус: ідентифікатор, координати, висоту, заряд батареї та стан.

Створіть новий проєкт для цієї програми:

```
cargo new drone_status
cd drone_status
```

Відкрийте `src/main.rs` і замініть вміст на наступний код. Програма моделює виведення діагностичної інформації бортового комп'ютера БПЛА — ту саму інформацію, яку бачить оператор на наземній станції управління.

```rust
fn main() {
    // === Ідентифікація агента ===
    let drone_id = "БПЛА-07";
    let mission = "Патрулювання периметра";

    // === Навігаційні дані ===
    let latitude = 50.450001;
    let longitude = 30.523410;
    let altitude = 120.5; // метрів над рівнем моря

    // === Стан системи ===
    let battery = 73.8; // відсотків
    let speed = 12.4;   // м/с
    let temperature = 22.3; // градусів Цельсія, температура двигуна

    // === Вивід статусу ===
    println!("╔══════════════════════════════════════╗");
    println!("║       СТАТУС БПЛА-АГЕНТА             ║");
    println!("╠══════════════════════════════════════╣");
    println!("║ Агент:    {}", drone_id);
    println!("║ Місія:    {}", mission);
    println!("╠══════════════════════════════════════╣");
    println!("║ Широта:   {:.6}°", latitude);
    println!("║ Довгота:  {:.6}°", longitude);
    println!("║ Висота:   {:.1} м", altitude);
    println!("║ Швидкість: {:.1} м/с", speed);
    println!("╠══════════════════════════════════════╣");
    println!("║ Батарея:  {:.1}%", battery);
    println!("║ Темп. двигуна: {:.1}°C", temperature);
    println!("╚══════════════════════════════════════╝");

    // Попередження, якщо заряд нижче порогу
    // (поки що — просто текст, без умовних операторів)
    println!();
    println!("Примітка: поріг повернення — 30%. Запас є.");
}
```

Запустіть:

```
cargo run
```

Вивід:

```
╔══════════════════════════════════════╗
║       СТАТУС БПЛА-АГЕНТА             ║
╠══════════════════════════════════════╣
║ Агент:    БПЛА-07
║ Місія:    Патрулювання периметра
╠══════════════════════════════════════╣
║ Широта:   50.450001°
║ Довгота:  30.523410°
║ Висота:   120.5 м
║ Швидкість: 12.4 м/с
╠══════════════════════════════════════╣
║ Батарея:  73.8%
║ Темп. двигуна: 22.3°C
╚══════════════════════════════════════╝

Примітка: поріг повернення — 30%. Запас є.
```

Розберемо, що тут відбувається.

Змінні створені через `let` і зберігають різні типи даних: текст (`"БПЛА-07"`), цілі числа, дробові числа. Rust автоматично визначає тип кожної змінної за значенням — це називається type inference (виведення типу). Детально про типи та змінні ми поговоримо у Розділі 5.

Координати виводяться з точністю шість знаків (`{:.6}`) — стандартна навігаційна точність GPS. Висота та швидкість — з одним знаком (`{:.1}`), бо більша точність для відображення оператору не потрібна.

Символи `╔`, `║`, `╠`, `╚` — це Unicode-символи рамки. Rust працює з Unicode нативно, тому будь-які символи, включаючи кирилицю та спецсимволи, виводяться без проблем.

Коментар `// (поки що — просто текст, без умовних операторів)` — це чесне зауваження. Зараз програма завжди виводить "Запас є", навіть якщо батарея на нулі. У Розділі 6 ви навчитесь умовним операторам і зможете виводити різні повідомлення залежно від рівня заряду. Саме так працює еволюція проєкту: кожен розділ знімає обмеження попередньої реалізації.

## 3.9. Антипатерни: як НЕ треба

Кілька типових помилок, які роблять початківці.

**Неправильно — забута крапка з комою:**

```rust
fn main() {
    println!("Привіт")  // ПОМИЛКА: немає ;
}
```

Помилка компілятора:

```
error: expected `;`, found `}`
```

Компілятор чітко каже: очікував крапку з комою, а знайшов закриваючу дужку. Виправлення — додати `;` після `println!("Привіт")`.

**Неправильно — невідповідність кількості заповнювачів:**

```rust
fn main() {
    // Два заповнювачі, але три значення — ПОМИЛКА
    println!("{} та {}", 1, 2, 3);
}
```

Помилка:

```
error: argument never used
```

Компілятор повідомляє: третій аргумент (`3`) ніде не використовується. Виправлення: або додати `{}` у рядок, або прибрати зайве значення.

**Неправильно — одинарні лапки для рядка:**

```rust
fn main() {
    println!('Привіт'); // ПОМИЛКА: одинарні лапки
}
```

У Rust одинарні лапки використовуються для окремих символів (`'A'`, `'Б'`, `'7'`), а не для рядків. Рядки — завжди в подвійних лапках: `"Привіт"`.

## Troubleshooting

**`error: expected ';', found '}'`**

Ви забули крапку з комою наприкінці рядка. У Rust кожна інструкція закінчується `;`. Найчастіше це трапляється після `println!(...)`. Додайте `;`.

**`error: argument never used`**

У `println!` більше значень, ніж заповнювачів `{}`. Або додайте `{}` у рядок форматування, або приберіть зайвий аргумент.

**`error: invalid reference to positional argument 0 (no arguments were given)`**

У рядку форматування є `{}`, але після рядка немає значення для підстановки. Додайте значення після коми.

**`error: 1 positional argument in format string, but 2 arguments were given`**

Один `{}` у рядку, а після коми — два значення. Або додайте другий `{}`, або приберіть зайве значення.

**`error: unknown character escape: '\a'`**

Ви використали зворотний слеш з символом, який не є допустимою escape-послідовністю. Допустимі: `\n` (новий рядок), `\t` (табуляція), `\\` (слеш), `\"` (лапка), `\0` (нульовий символ). Якщо хочете вивести просто `\` — пишіть `\\`.

**Текст кирилицею виводиться коректно, але рамка "з'їхала".**

Unicode-символи рамки (`╔`, `║`, `═`) займають різну ширину в різних терміналах та шрифтах. Це не помилка вашого коду — це особливість відображення Unicode. У реальному проєкті для форматованого виводу використовують спеціальні бібліотеки. Для навчання — рамка не обов'язкова, звичайного тексту достатньо.

**Попередження `warning: unused variable: 'x'`**

Ви створили змінну через `let`, але ніде її не використали. Це не помилка (програма скомпілюється), але Rust попереджає: навіщо змінна, яка не використовується? Або використайте її в `println!`, або видаліть. Якщо змінна потрібна тимчасово — додайте підкреслення перед ім'ям: `let _x = 5;` — це скаже компілятору "я знаю, що вона не використана".

## Додатково

### Інші можливості форматування

Окрім `{}` і `{:.N}`, Rust підтримує багато інших форматів. Ось кілька, що можуть знадобитися:

```rust
fn main() {
    // Ширина поля — мінімальна кількість символів
    println!("[{:10}]", "test");     // [test      ]  — вирівняно ліворуч
    println!("[{:>10}]", "test");    // [      test]  — вирівняно праворуч
    println!("[{:^10}]", "test");    // [   test   ]  — по центру

    // Заповнення нулями
    println!("{:05}", 42);           // 00042

    // Двійковий, вісімковий, шістнадцятковий формати
    println!("Десятковий: {}", 255);
    println!("Двійковий:  {:b}", 255);
    println!("Вісімковий: {:o}", 255);
    println!("Шістнадцятковий: {:x}", 255);
    println!("Шістнадцятковий: {:X}", 255);
}
```

Вивід:

```
[test      ]
[      test]
[   test   ]
00042
Десятковий: 255
Двійковий:  11111111
Вісімковий: 377
Шістнадцятковий: ff
Шістнадцятковий: FF
```

Вам не потрібно запам'ятовувати все це зараз. Коли знадобиться — ви знатимете, що таке можливо, і знайдете синтаксис у документації або через AI.

### print! vs println! vs eprint! vs eprintln!

Окрім `print!` та `println!`, є ще `eprint!` та `eprintln!`. Різниця: `print!` виводить у стандартний потік виводу (stdout), а `eprint!` — у стандартний потік помилок (stderr). Для людини, що дивиться на термінал, різниці немає — текст з'являється на екрані. Але коли вивід програми перенаправляється у файл, stderr все одно виводиться на екран. Це корисно для повідомлень про помилки — вони не "загубляться" у файлі з нормальним виводом.

## Резюме

Кожна програма на Rust починається з функції `fn main()` — це точка входу, яку шукає операційна система при запуску.

`println!` — макрос виводу тексту з переходом на новий рядок. Знак `!` означає макрос, а не функцію. `print!` робить те саме без переходу на новий рядок.

Заповнювачі у рядку форматування: `{}` для стандартного виводу, `{:.N}` для дробових чисел з заданою точністю, `{:?}` для налагоджувального виводу.

Кожна інструкція в Rust закінчується крапкою з комою `;`. Забута крапка з комою — найчастіша помилка першого тижня.

Коментарі: `//` — однорядковий, `/* */` — багаторядковий, `///` — документаційний. Хороший коментар пояснює *чому*, а не *що*.

Escape-послідовності: `\n` — новий рядок, `\t` — табуляція, `\\` — зворотний слеш, `\"` — лапка.

Кількість `{}` у рядку форматування повинна збігатися з кількістю значень. Rust перевіряє це під час компіляції.
