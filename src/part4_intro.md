# ЧАСТИНА IV: SMART POINTERS ТА СПІЛЬНА ПАМ'ЯТЬ

## Стан агента на початку Частини IV

Після Частини III ваш агент — робастна система. Динамічні колекції (Vec, HashMap, HashSet) зберігають історію, об'єкти та зони. Ієрархія помилок через thiserror забезпечує graceful degradation замість паніки. Trait Agent дозволяє створювати гетерогенний рій через trait objects. Ітератори обробляють потоки даних декларативно.

Але агент працює в одному потоці. Рій із десяти агентів виконується послідовно: поки SCOUT-01 сканує зону, інші дев'ять чекають. У реальному світі кожен БПЛА — окремий процес, що діє паралельно. Координатор рою не чекає відповіді від кожного агента по черзі — він надсилає команди одночасно та обробляє відповіді міру надходження. Для цього потрібна багатопотоковість.

Частина IV будує фундамент для паралельного рою у вісім кроків. Closures (Розділ 31) — анонімні функції, що захоплюють оточення; вони стануть "мозком" кожного потоку. Box<T> (Розділ 32) — явне розміщення на купі для рекурсивних типів та trait objects. Rc та RefCell (Розділ 33) — спільне володіння та внутрішня мутабельність для однопотокового коду. Lifetimes (Розділ 34) — анотації часу життя, які компілятор потребує для гарантій безпеки. Serde (Розділ 35) — серіалізація стану агента для збереження та обміну повідомленнями. Threads (Розділ 36) — справжня паралельність: кожен агент у своєму потоці. Arc та Mutex (Розділ 37) — thread-safe спільний стан: карта світу, до якої звертаються всі агенти. Channels (Розділ 38) — обмін повідомленнями між потоками без спільного стану.

PE-фокус зміщується до роботи зі складними помилками компілятора (borrow checker у паралельному контексті) та проєктування паралельних архітектур з допомогою AI.

Еволюція агента: з автономного об'єкта (Частина II) та робастної системи (Частина III) агент стає процесом — елементом паралельного рою, що працює одночасно з іншими агентами, ділиться даними через захищені механізми та обмінюється повідомленнями через канали.
