# ЧАСТИНА VI: АРХІТЕКТУРА, МАКРОСИ ТА ЕКОСИСТЕМА

## Стан агента на початку Частини VI

Після Частини V ваш рій — повноцінна асинхронна мультиагентна система. Актори-агенти працюють конкурентно на Tokio runtime, комунікують через mpsc/broadcast/watch канали, обчислюють маршрути паралельно через Rayon, логують через tracing. Архітектура Actor Model ізолює стан кожного компонента. Graceful shutdown координує завершення.

З точки зору функціональності — система готова. Але з точки зору професійної розробки — є прогалини. Повторюваний boilerplate при створенні нових акторів (enum Message, struct Actor, Handle, new, run — кожен раз однаковий каркас). Відсутність інструментів кодогенерації (макроси). Незнання меж safe Rust та коли потрібен unsafe. Поверхневе знайомство з просунутими PE-техніками. Відсутність мережевої комунікації для розподіленого рою.

Частина VI закриває ці прогалини у п'яти розділах. Декларативні макроси (Розділ 46) — macro_rules! для скорочення boilerplate. Процедурні макроси (Розділ 47) — derive macros для автоматичної генерації коду. Unsafe Rust (Розділ 48) — межі безпеки, raw pointers, FFI. Просунутий Prompt Engineering (Розділ 49) — chain-of-thought, few-shot, професійний workflow з AI. Мережева комунікація та екосистема (Розділ 50) — TCP, розподілений рій, ресурси для подальшого навчання.

PE-фокус: AI стає повноцінним інструментом — генерація boilerplate, макросів, документації, тестів.

Еволюція агента: від коду до архітектурної системи з кодогенерацією, мережею та професійними практиками.
