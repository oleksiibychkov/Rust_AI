## Додаток C. Cargo та інструменти

Основні команди:

- `cargo new project_name` — створити проєкт
- `cargo build` / `cargo build --release` — компіляція
- `cargo run` — компіляція + запуск
- `cargo test` — запуск тестів
- `cargo clippy` — лінтер (рекомендації щодо стилю та потенційних помилок)
- `cargo fmt` — автоматичне форматування
- `cargo doc --open` — генерація та відкриття документації
- `cargo expand` — перегляд розгорнутих макросів
- `cargo audit` — перевірка вразливостей у залежностях
- `cargo bench` — бенчмарки (з criterion)

Корисні інструменти:

- `rust-analyzer` — LSP-сервер для IDE (автодоповнення, go-to-definition)
- `miri` — інтерпретатор для перевірки unsafe коду на UB
- `cargo-watch` — автоматичний rebuild при зміні файлів
- `cargo-deny` — перевірка ліцензій залежностей

## Додаток D. Глосарій

| Термін (укр.) | Термін (англ.) | Визначення |
|--------------|----------------|------------|
| Володіння | Ownership | Кожне значення має рівно одного власника |
| Позичання | Borrowing | Тимчасовий доступ до значення через & або &mut |
| Час життя | Lifetime | Область коду, де посилання гарантовано валідне |
| Трейт | Trait | Іменований набір методів — контракт поведінки |
| Узагальнення | Generics | Параметризація коду типом |
| Замикання | Closure | Анонімна функція, що захоплює оточення |
| Розумний вказівник | Smart pointer | Тип з ownership-семантикою (Box, Rc, Arc) |
| Потік | Thread | Незалежна послідовність інструкцій процесора |
| Асинхронність | Async | Модель конкурентності з призупиненням при IO |
| Актор | Actor | Ізольована одиниця обчислення з mailbox |
| Канал | Channel | Механізм передачі повідомлень між потоками/tasks |
| Серіалізація | Serialization | Перетворення структури у байти для збереження/передачі |
| Макрос | Macro | Код, що генерує код при компіляції |
| Небезпечний | Unsafe | Блок де компілятор не перевіряє 5 операцій |
| Промпт | Prompt | Текстовий запит до AI-асистента |
| Галюцинація | Hallucination | Впевнена але неправильна відповідь AI |
